# caffeinate-ui

A macOS menubar app that keeps your Mac awake. It wraps the built-in
`caffeinate` command with duration presets, a live countdown, and per-sleep-type
control.

No dependencies, no Xcode required.

## Requirements

- macOS 13 or later
- Swift toolchain (Xcode **or** just the Command Line Tools: `xcode-select --install`)

## Install

```bash
git clone https://github.com/narate/caffeinate-ui.git
cd caffeinate-ui
./scripts/bundle.sh
open build/caffeinate-ui.app
```

To keep it around, drag `build/caffeinate-ui.app` to `/Applications`.

Gatekeeper won't complain: a binary you compiled locally never gets the
`com.apple.quarantine` attribute, which is only applied to downloads.

For development, `swift run caffeinate-ui` skips the bundle step.

## Usage

Click the cup in the menubar:

```
☕ 42m              icon plus countdown; icon only when idle
─────────────
  Indefinite         reads "Turn Off" while a session is running
  15 minutes
  1 hour
  2 hours
─────────────
  Prevent ▸  ✓ Display (-d)
             ✓ Idle (-i)
               Disk (-m)
               System (-s, AC power only)
─────────────
  Launch at Login
─────────────
  Quit
```

**Prevent** maps to `caffeinate`'s own flags. The defaults, display and idle
sleep, are what "keep awake" means for most uses. Changing them mid-session
restarts `caffeinate` without disturbing your countdown. `-s` only takes effect
on AC power.

**Launch at Login** appears only in the bundled `.app` — under `swift run` there
is no bundle for macOS to register, so the option is hidden rather than offered
and silently ignored.

## How it works

The app spawns `/usr/bin/caffeinate` as a child process and terminates it to
release. It doesn't reach past `caffeinate` to IOKit power assertions —
`caffeinate` is the platform feature that already wraps those.

The child is spawned watching the app's own PID:

```
caffeinate -d -i -w <our pid>
```

`-w` makes `caffeinate` exit when the watched process exits. So if the app
crashes or is force-quit, the child goes with it. Without that, a crash would
strand a `caffeinate` process holding your Mac awake with no UI left to stop it.

Expiry is driven by the app's own timer rather than `caffeinate -t`, so there is
exactly one source of truth for when a session ends — the countdown you see
can't disagree with the assertion actually being held.

## Development

```bash
swift build                          # debug build
swift run caffeinate-ui              # run without bundling
swift run caffeinate-ui --self-check # run the assertions
./scripts/bundle.sh                  # release build, icon, ad-hoc sign, .app
```

`--self-check` is the test suite. There is no test target, because neither
`XCTest` nor `Testing` can be imported with only the Command Line Tools
installed — both ship inside Xcode. Assertions use `precondition` rather than
`assert`, since `assert` is compiled out under `-O`; `bundle.sh` runs the
self-check against the *release* binary specifically to prove the checks
survived optimization.

The app icon is generated by `scripts/make-icon.swift` at bundle time rather
than committed, so it cannot drift from the code that draws it. It's drawn from
Core Graphics paths rather than reusing the menubar's SF Symbol, because Apple's
SF Symbols license doesn't permit symbols in app icons.

Design notes and the implementation plan live in [`docs/superpowers/`](docs/superpowers/).
